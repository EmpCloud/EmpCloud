name: Always get approval before building
description: User requires explicit approval before writing any code or making changes, especially during planning stages. Always show plans/READMEs first and wait for confirmation.
type: feedback
---

Always show the user what you're going to do and get explicit approval before starting to build or write code.

**Why:** User wants to review and approve all plans, READMEs, and implementation details before any code is written. They don't want surprises — especially at the planning stage.

**How to apply:** During planning phases, always present the full content (README, plan, architecture) to the user first. Wait for their "go ahead" or approval before executing. Never assume approval. This applies to READMEs, implementation plans, code scaffolding — everything.
